Task Type:
Full Stack

Task Coding Language:
TypeScript / React + Node.js


Task description:
Build a paper submission and peer review system for academic conferences. Conference chairs create a conference with a name, description, important dates (submission deadline, notification date, camera-ready deadline), topic areas, and submission guidelines. Authors submit papers by uploading a PDF file with a title, abstract, author list with names, affiliations, and email addresses, topic area selections, and optional keywords. The system generates a unique paper ID and confirmation. During the review phase, the program committee chair assigns papers to reviewers — registered reviewers indicate their topic area expertise and declare conflicts of interest with specific authors. An automatic assignment suggestion algorithm matches papers to reviewers based on topic area overlap while avoiding conflicts and balancing the review load evenly. Reviewers access their assigned papers through a reviewer dashboard, reading the PDF and submitting a review form with scores on multiple criteria (originality, technical quality, clarity, relevance) each rated 1-5, an overall recommendation of accept, weak accept, weak reject, or reject, a detailed text review with feedback for the authors, and a confidential note visible only to the chair. The chair sees all reviews per paper in a decision dashboard, makes final accept/reject decisions informed by reviewer scores, and can see aggregate statistics like acceptance rate and score distributions. Authors receive notification with the decision and anonymized review text through the system. A rebuttal phase optionally lets authors respond to reviews before final decisions. Camera-ready submission allows accepted authors to upload a revised version. A conference program builder lets the chair organize accepted papers into sessions with time slots and room assignments. All data persists in SQLite with JWT authentication supporting chair, reviewer, and author roles.
